# 第一版需求基线

## 交易范围

- 交易所：Bitget
- 产品：U 本位永续合约
- 方向：做多和做空
- 保证金模式：全仓（纸面模拟）
- 运行环境：先在个人 Windows 电脑运行，后续迁移服务器
- 执行方式：完全自动，先使用 Bitget 实盘公开行情驱动的本地纸面交易
- 纸面交易阶段不配置 API Key，不允许调用任何私有下单接口

## 币种池

- 上线至少 30 天
- 24 小时 USDT 成交额不少于 1,000 万
- 买卖价差低于 0.3%
- 排除稳定币和成交异常币
- 每 5 分钟重新筛选

## 主策略

放量突破后，使用高周期趋势确认，并等待低周期回踩后入场。第一版只实现这一套可归因策略，同时支持多空。

持仓目标为数小时。退出机制包括 ATR 初始止损、达到 1R 后移动到保本、分批止盈以及 ATR 移动止损。

## 风控

- 每笔最多承担账户权益 1% 的风险
- 每日亏损达到 5% 后，当日禁止新开仓
- 权益从高水位回撤 15% 后进入只平仓模式
- 流动性不足、连续亏损或交易所接口异常时可自动进入只平仓模式
- 新仓使用 Bitget 对相应合约公布的最高杠杆，但仓位数量仍按止损距离和单笔风险计算
- 排除 Bitget 标记的 RWA、股票及商品映射合约
- 测试阶段不因连续亏损自动暂停开仓
- 达到 2R 后止盈三分之一，剩余仓位使用 3ATR 跟踪止损
- 收盘重新进入突破区间，或 6 根 5 分钟 K 线仍未达到 0.5R 时提前退出
- 测试资金阶段最多持有 3 个币；正式阶段最多 5 个
- 单个币种累计开放风险不超过权益 2%
- 仅在已有仓位盈利且总风险仍合规时允许顺势加仓

## 功能范围

- 自动选币、信号、下单与订单管理
- 回测、手续费、滑点和资金费率建模
- 仓位及账户级风险管理
- 邮件告警
- 本机 Web 控制台
- SQLite 开发存储，保留迁移 PostgreSQL 的接口
- 交易日志、收益报表、断线恢复

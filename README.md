# Bitget 山寨币趋势交易系统

面向 Bitget U 本位永续合约的自动化量化交易项目。第一版聚焦：

- 5 分钟扫描合约币种池
- 放量突破 + 多周期趋势确认 + 回踩入场
- 多空双向、逐仓模式
- ATR 止损、1R 保本、分批止盈、移动止损
- 账户级风险熔断和“只平仓”安全模式
- 默认使用本地纸面交易，实盘必须显式解锁

> 本项目是交易软件，不承诺盈利。模拟盘和历史回测不能代表实盘结果。

## 已确认的风险边界

- 单笔账户风险：1%
- 单日最大亏损：5%，触发后暂停开仓至下一交易日
- 最大回撤：15%，触发后进入只平仓模式，需人工解除
- 动态杠杆：2–5 倍
- 只允许盈利后的顺势加仓，禁止亏损补仓
- 测试资金阶段最多同时持有 3 个币，正式资金可调整到 5 个
- 单币累计风险不超过账户权益的 2%
- 禁止马丁格尔、网格摊平和逆势抄底

## 快速开始

需要 Python 3.11 或更高版本。

```powershell
Copy-Item .env.example .env
notepad .env
.\run.ps1
```

在 Windows 命令提示符（CMD）中可以直接运行：

```cmd
run.cmd
```

单次扫描和邮件测试：

```cmd
run.cmd scan
run.cmd test-email
```

启动本机 Web 控制台：

```cmd
web.cmd
```

然后访问 `http://127.0.0.1:8000`。交易扫描器和 Web 控制台需要分别打开一个 CMD 窗口运行。

如果 PowerShell 首次阻止脚本执行，可使用：

```powershell
powershell -ExecutionPolicy Bypass -File .\run.ps1
```

纸面交易直接读取 Bitget 实盘公开行情，在本地模拟滑点、手续费、持仓和盈亏，不需要 API Key，也不会向交易所发送订单。

QQ 邮件需要在 QQ 邮箱中开启 SMTP 服务并创建授权码。`.env` 中的 `SMTP_PASSWORD` 应填写授权码，不是 QQ 登录密码。
